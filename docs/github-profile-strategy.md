# GitHub Profile Strategy

Manuel's portfolio should be the primary public source of professional credibility. GitHub should support that positioning through selected public work, experiments and current technical interests.

## Context

Most recent professional work since 2021 has happened through private enterprise repositories, consulting engagements and organization-managed GitHub accounts tied to work email identities. That is normal for enterprise and client work, and it should not be treated as a gap.

## Strategy

- Keep work GitHub accounts separate when they are tied to enterprise or client access.
- Do not attempt to merge, expose or recreate private enterprise contribution activity.
- Add only verifiable personal emails to the personal GitHub account.
- Use the personal GitHub as a curated public portfolio, not a complete employment record.
- Update the GitHub bio to match the portfolio positioning.
- Pin only modern, relevant repositories that support the Senior Frontend Engineer narrative.

## Recommended Public Projects

Build or polish two to three strategic projects:

1. AI streaming demo with Next.js.
2. Component system playground with Storybook.
3. Frontend architecture notes covering micro frontends, feature flags, design systems, CI/CD and modernization.

## Pinning Guidance

Prioritize:

- Portfolio source code.
- AI streaming or AI UX experiments.
- Modern Next.js/React/TypeScript projects.
- Component system or Storybook examples.
- Architecture notes with clear README files.

Avoid pinning:

- Old tutorial projects.
- Bootcamp/student-style demos.
- Repositories that do not represent current senior-level work.
- Forks or copied repos without clear context.

## Suggested GitHub Bio

Senior Frontend Engineer focused on React, Next.js, micro frontends, design systems, AI product UX and enterprise frontend modernization.
